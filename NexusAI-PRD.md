# NexusAI 产品需求文档 (PRD)

> **产品名称**: NexusAI — 金融行业工作流数字孪生平台
> **版本**: v1.0
> **日期**: 2026-05-25
> **状态**: 设计中

---

## 一、问题陈述 (Problem Statement)

金融机构面临三大核心痛点：

1. **信息不对称与决策滞后**：银行、券商、基金等机构中，管理者无法实时感知团队工作状态。项目延期、资源错配、协作瓶颈通常在复盘时才被发现，错失最佳干预窗口。5000人的大型金融机构中，跨部门协作依赖邮件和会议，信息传递平均延迟 4-6 小时。

2. **资源利用率黑盒**：团队负载和产能无法量化。常出现"A团队加班到崩溃，B团队却在等需求"的局面。传统项目管理工具（Jira、禅道）只记录任务状态，无法分析协作效率、预测瓶颈、推演资源调整的影响。

3. **合规与风控压力**：金融行业面临等保三级、GDPR等严苛监管要求。任何数据处理和AI决策都必须可审计、可追溯。现有AI方案多为黑盒模型，无法满足监管审查需求。

**NexusAI 的解法**：将金融机构的"人-任务-数据-流程"全部映射到3D虚拟空间，让AI实时预测瓶颈、优化资源、预警风险，把隐性协作关系变成显性数字孪生。

---

## 二、解决方案 (Solution)

NexusAI 是一个面向金融机构的**工作流数字孪生平台**，核心能力：

- **实时映射**：通过 API 集成 + AI 被动观察 + 主动录入，将员工、任务、文档、沟通等全部工作要素同步到数字孪生空间
- **3D 可视化**：以"虚拟办公大楼"形态呈现整个机构的工作流状态，支持三层钻取（部门楼层 → 项目视图 → 工作流管道）
- **AI 预测引擎**：瓶颈预测、资源优化、协作效率分析、智能决策推演
- **多智能体协同**：每个员工、部门、项目配备专属 AI Agent，按合同网协议协同决策
- **金融级合规**：等保三级、GDPR、区块链审计追踪、联邦学习数据不出域

## 三、用户故事 (User Stories)

### 3.1 高管视角

1. 作为银行行长，我想要在3D虚拟大楼中一眼看到全行各部门的实时负载状态，以便快速定位瓶颈部门
2. 作为CIO，我想要系统自动推送"12个项目中3个有延期风险"的预警，以便提前调配资源
3. 作为风控总监，我想要查看任何历史决策的完整审计链路（谁、何时、基于什么数据做出的决策），以便通过监管检查
4. 作为CEO，我想要运行"What-If"推演（如"把华东团队20%人力调去华南项目会怎样"），以便做出数据驱动的决策
5. 作为董事会成员，我想要一份月度"组织健康度报告"（协作效率趋势、瓶颈热力图、资源利用率），以便评估管理效能

### 3.2 部门经理视角

6. 作为部门经理，我想要在虚拟楼层中看到每个团队成员的工作负载饱和度，以便合理分配新任务
7. 作为项目经理，我想要看到项目关联的所有人员、文档、会议的实时联动状态，以便跟踪项目健康度
8. 作为技术负责人，我想要系统识别出"张三和李四分别在做同一件事"的重复劳动，以便消除浪费
9. 作为业务线负责人，我想要AI自动建议"这个审批卡在王五那里3天了，建议升级到李总"，以便加速决策
10. 作为团队Leader，我想要看板自动按优先级排序，并用颜色标注风险等级，以便聚焦关键任务

### 3.3 员工视角

11. 作为普通员工，我想要我的个人 Agent 自动帮我分类待办、提醒截止日期，以便专注重要工作
12. 作为分析师，我想要系统自动从 Wind 万得中提取我关注的数据变化并推送摘要，以便第一时间反应
13. 作为交易员，我想要 Agent 在关键风险指标突破阈值时毫秒级弹窗预警，以便及时止损
14. 作为研究员，我想要知识图谱自动关联我阅读的报告与团队的知识库，以便发现交叉洞察
15. 作为新入职员工，我想要虚拟办公室帮我可视化"遇到XX问题该找谁"，以便快速融入组织

### 3.4 合规与风控视角

16. 作为合规官，我想要所有AI决策都有区块链存证（决策输入、模型版本、推理路径、输出结果），以便审计
17. 作为风控经理，我想要实时监控跨部门协作中的合规风险（如"研究员A和交易员B是否违反信息隔离墙"），以便及时阻断
18. 作为审计人员，我想要按时间轴回放任意项目从立项到交付的完整工作流，以便追溯问题
19. 作为数据保护官，我想要确认所有涉及个人隐私的处理均已完成差分隐私脱敏，以便满足GDPR要求
20. 作为安全负责人，我想要查看联邦学习各参与节点的梯度交换记录，以便确认数据未出域

### 3.5 系统管理视角

21. 作为IT管理员，我想要一键切换本地/混合云渲染模式，以便保证弱网环境的可用性
22. 作为系统运维，我想要监控三级AI推理（边缘/区域/云端）的延迟和准确率，以便优化算力分配
23. 作为数据工程师，我想要可视化各数据源（Wind/恒生/企微/Jira）的接入状态和同步延迟，以便排查数据问题

## 四、实现决策 (Implementation Decisions)

### 4.1 架构决策

**五层技术架构**：

| 层级 | 职责 | 关键技术选型 |
|------|------|-------------|
| 体验编排层 | 角色驱动的3D虚拟办公室 | 五大角色面板→3D虚拟办公室三层映射 |
| 决策智能层 | 三级AI推理引擎 | 边缘量化模型 / 区域GPT-4+RAG / 云端联邦学习 |
| 仿真环境层 | 混合渲染引擎 | Three.js WebGL/WebGPU双后端 + 混合云渲染 |
| 行为建模层 | 金融智能模型 | GNN协作分析 + Transformer任务识别 + FIBO知识图谱 |
| 数据织物层 | 全域数据接入 | LakeHouse + Flink CDC + 同态加密/差分隐私 |

**三级AI推理架构**：

| 层级 | 延迟 | 模型 | 场景 |
|------|------|------|------|
| 边缘（本地） | <100ms | 量化模型 + 规则引擎 | 个人任务提醒、风险阈值告警 |
| 区域（集群） | <5s | GPT-4 + RAG + 多Agent协同 | 资源优化、协作分析、瓶颈识别 |
| 云端（万卡） | <1h | 联邦学习 + 强化学习 | 战略决策推演、组织优化仿真 |

**混合云部署**：敏感数据（交易记录、客户信息）本地处理，脱敏数据（协作统计、负载指标）上云渲染。等保三级合规，GDPR就绪。

**3D渲染策略**：WebGPU优先（性能为WebGL的2-10倍），WebGL兜底。日常20-50人客户端渲染，重要汇报100+云端推流。弱网自动降级2D模式。

### 4.2 模块设计

**核心模块**：

1. **数据接入模块**：对接 Wind 万得（国内90%+金融机构在用）、恒生电子（证券/基金核心系统80%+）、企微/飞书/钉钉/Teams、Jira、邮件系统。采用 Flink CDC 实时同步 + LakeHouse 统一存储。

2. **行为建模模块**：GNN 图神经网络分析跨部门协作关系，Transformer 识别任务执行模式，FIBO（金融行业业务本体）构建行业知识图谱，实现语义级理解。

3. **仿真渲染模块**：Three.js 引擎驱动，WebGL/WebGPU 双后端自适应。生成式3D建模（AI生成粗模+人工精修），支持部门楼层、项目视图、工作流管道三层钻取。

4. **AI决策模块**：三级推理架构。多智能体系统——员工Agent（个人数字孪生，毫秒级）、部门Agent（团队数字孪生，秒级）、项目Agent（全链路仿真，小时级）。协同机制：合同网协议（任务分配）+ VCG拍卖（资源配置）+ 市场博弈（复杂决策）。

5. **安全合规模块**：联邦学习（各机构本地训练，只传梯度，数据不出域）、同态加密+差分隐私（数据处理）、区块链审计（决策存证）、RBAC五角色权限体系。

### 4.3 权限模型

| 角色 | 可见范围 | 操作权限 | 3D视图 |
|------|----------|----------|--------|
| 高管 | 全机构 | 查看所有、决策推演、导出报告 | 全局大楼视图 |
| 部门经理 | 本部门+关联项目 | 资源调整、任务分配、查看报表 | 部门楼层+项目视图 |
| 员工 | 个人任务+参与项目 | 任务更新、状态录入、查看个人分析 | 个人工作站+团队视图 |
| 合规官 | 全机构（只读） | 审计查询、合规检查、日志导出 | 合规审查面板 |
| 风控官 | 全机构（只读+阻断） | 风险预警、流程阻断、合规审查 | 风控中心面板 |

### 4.4 商业化策略

- **私有化授权费**：50万-500万（按机构规模和功能模块）
- **年服务费**：授权费的15%-20%（含更新、维护、AI模型迭代）
- **灯塔客户策略**：国有大行 → 头部券商 → 城商行/基金 → 中小机构，逐级渗透
- **差异化壁垒**：金融专用FIBO本体、多智能体协同、三级隐私架构、渐进式3D

## 五、测试策略 (Testing Decisions)

### 5.1 测试原则

只测试外部可观测行为，不测试内部实现细节。测试应验证：给定输入X，系统输出Y，且行为可审计。

### 5.2 测试范围

| 模块 | 测试重点 | 测试方法 |
|------|----------|----------|
| 数据接入 | CDC同步延迟、数据完整性、异构系统适配 | 集成测试 + 压力测试 |
| 行为建模 | GNN协作分析准确率、任务模式识别召回率、FIBO本体推理 | 单元测试 + A/B对照 |
| 仿真渲染 | WebGL/WebGPU兼容性、3D帧率（目标60fps）、弱网降级 | 兼容性矩阵 + 性能基准 |
| AI决策 | 瓶颈预测提前量、资源优化建议接受率、多Agent协同死锁 | 仿真测试 + 人工评估 |
| 安全合规 | 联邦学习梯度不漏数据、区块链审计完整性、权限隔离 | 渗透测试 + 审计验证 |

### 5.3 性能基准

| 指标 | 目标值 | 测试环境 |
|------|--------|----------|
| 边缘推理延迟 | <100ms | 本地RTX 5080 |
| 区域推理延迟 | <5s | 集群GH200 |
| 3D帧率（50人场景） | ≥60fps | WebGPU + RTX 5080 |
| 3D帧率（100+推流） | ≥30fps | 云端GPU推流 |
| 数据同步延迟 | <1min | Flink CDC |
| 审计查询响应 | <3s | 区块链索引 |

## 六、范围外 (Out of Scope)

1. **非金融行业定制**：v1.0聚焦金融行业通用场景，不包含制造业、零售、医疗等行业定制
2. **移动端原生App**：v1.0仅支持浏览器（WebGL/WebGPU）+桌面客户端（Unity），移动端仅支持2D轻量视图
3. **实时交易风控**：NexusAI定位为工作流数字孪生，不替代专业的实时交易风控系统（但可与之集成）
4. **HR系统**：不包含招聘、绩效考核、薪酬等HR职能（但可对接HR系统获取组织架构）
5. **财务系统**：不包含账务处理、报表生成等财务职能（但可对接财务系统获取成本数据）
6. **自研大模型**：不包含从头训练大语言模型，而是基于现有模型（GPT-4等）做金融领域微调和RAG增强
7. **IoT设备管理**：不涉及物理传感器、门禁、摄像头等IoT设备的管理（但预留扩展接口）

## 七、补充说明 (Further Notes)

### 7.1 技术风险

- **联邦学习性能**：梯度交换的通信开销可能影响训练效率，需在隐私保护与模型精度间做权衡
- **生成式3D质量**：当前生成式3D定位"从0到0.6"，AI生成粗模后仍需人工精修，这是行业共识
- **多Agent死锁**：合同网协议+VCG拍卖在复杂博弈场景下可能出现死锁或振荡，需要超时熔断机制

### 7.2 市场风险

- **客户教育成本**：数字孪生概念在金融行业较新，需要标杆案例证明ROI
- **数据接入阻力**：金融机构系统封闭，API对接可能遇到技术壁垒和合规阻力
- **竞品动向**：需持续关注Kyndryl、西门子等巨头的金融行业布局
