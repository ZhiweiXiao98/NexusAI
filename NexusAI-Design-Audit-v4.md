# NexusAI 原型 v4 设计审计与方向报告

> **日期**: 2026-05-26
> **来源**: v3 原型深度分析 + 三个标杆案例研究
> **目标**: 为 v4 原型重写提供明确、可执行的设计方向

---

## 第一部分：标杆分析

### 1.1 腾讯 Marvis 3D 虚拟办公室

**为什么它会让人"笑起来"？**

Marvis 的 3D 虚拟办公室之所以有感染力，核心在于将抽象的 AI 工作可视化成了具象的、有生命感的"小黑马"角色。它不是冷冰冰的仪表盘，而是一个**有人情味的空间**。

| 维度 | 提炼 | 具体表现 |
|------|------|---------|
| 角色设计 | 小黑马是统一但各有差异的"员工"，每个对应一个 Agent（文件 Agent、搜索 Agent 等）。圆润、简洁、有辨识度。 | 关键：不是写实人物也不是抽象球体，而是处于"拟人化中间地带"——能承载性格，又不过于写实导致恐怖谷。 |
| 状态可视化 | 搬砖=工作中，打盹=空闲，冒烟=繁忙，欢呼=任务完成。状态用角色动画而非颜色表达。 | 关键：动画叙事 > 静态颜色。用户不需要读数值就能秒懂。 |
| 空间氛围 | 明亮的室内办公场景，木质地板+浅色墙面+现代家具。温暖但不"土"。 | 关键：白天室内自然光 + 软阴影，避免黑暗空洞。 |
| 色彩体系 | 低饱和度的暖灰 + 木质棕 + 少量品牌蓝。主色是环境色而非金色。 | 关键：色彩来自材质本身（木纹、白墙、绿植），不是人为的金色滤镜。 |
| 空间尺度 | 一个房间大小，用户一眼看完。每个 Agent 的工位是独立但相邻的。 | 关键：小空间 > 大场景。大场景必有空洞。 |

> **v4 核心启示**：金融行业需要自己的"角色语言"和"空间尺度"。小黑马的卡通感适合消费级 AI 助手，金融场景需要更克制的角色设计，但**空间尺度可以借鉴**。

### 1.2 Microsoft Smart Building Digital Twin

**专业又不冷感的现代办公室可视化**

Microsoft 的智慧楼宇数字孪生代表了企业级 3D 可视化的主流方向。核心设计原则：

| 维度 | 提炼 | v4 可借鉴 |
|------|------|----------|
| Fluent Design 原则 | 圆角几何、Acrylic 半透明材质、软阴影、微妙深度感 | 面板 UI 用玻璃拟态（Glassmorphism），但要冷色调而非金色 |
| 数据驱动极简主义 | 3D 场景干净简洁，颜色和动画只为突出洞察，不制造视觉噪音 | 办公室中数据只附着在相关物体上，不是到处漂浮 |
| LoD 分层 | 远观用低多边形模型保证性能，近看才加载高精模型 | 办公室场景本身就是近景，可用中等多边形，关键在材质和光影 |
| 空间 UI | 2D 面板 / 提示卡片放置在对应 3D 物体旁，有深度偏移和模糊背景 | KPI 卡片贴在工位上方、告警挂在会议室门口——数据属于空间 |
| 色彩系统 | 中性灰背景 + Fluent 蓝色交互元素 + 红/绿状态色。无金色。 | 主色：深蓝钢色（#1E3A5F），辅色：电光蓝（#3B82F6），底色：冷白/浅灰 |
| 预设视图 | "楼宇总览""楼层平面"等一键切换，以及从列表"飞到"房间 | v4 预设：全局俯视 / 员工工位 / 会议室 / 风控中心 |

> **v4 核心启示**：Microsoft 证明了"专业"和"有温度"不矛盾。方法是用材质本身（毛玻璃、磨砂金属）赋予质感，而不是用金色这种装饰色来制造"温暖假象"。**材质 > 颜色**。

### 1.3 优秀 3D 数据驾驶舱设计（Dribbble/Behance 高分作品）

**空间感和数据洞察的共存之道**

Dribbble/Behance 上的 3D 驾驶舱和数字孪生作品（如 HIGHTOPO、博维数孪案例）提供了空间嵌入数据的范式：

| 维度 | 提炼 | v4 可借鉴 |
|------|------|----------|
| 三层驾驶舱布局 | 顶部 KPI 仪表（全局指标）→ 中部 3D 场景 + 核心图表（空间映射）→ 底部明细（表格/列表） | 沿用 v3 的布局框架但需调整：顶部=全局 KPI 条，中部=3D 办公室主场景，右侧=详情面板 |
| 数据嵌入方式 | 高度/长度映射数值（柱体高低表示负载）；颜色渐变映射范围（蓝→绿→黄→红）；动效映射变化（粒子流表示数据流转） | 工位桌面高度可表示负载；会议室玻璃颜色表示占用状态；走廊地板光带表示工作流速 |
| 深色主题 + 科技蓝/青主色 | 70% 深色背景 + 青色/蓝色主色 + 1-2 个高饱和强调色用于告警 | v4 可以浅色为主（办公室自然光），数据层用深色卡片嵌入，形成反差 |
| 玻璃拟态（Glassmorphism） | 半透明毛玻璃面板 + 背景模糊 + 细边框 | 面板全部用玻璃拟态，底色冷白而非金色 |
| 流线/粒子数据可视化 | 流动线表示数据流向，避免静态图表 | 办公室的走廊、地板、天花板可以埋入数据流线，不抢眼但持续存在 |

> **v4 核心启示**：好的驾驶舱设计让**数据像是"长在"空间里**，而不是贴上去的标签。数据要有"物理锚点"——KPI 在工位上、风险在会议室玻璃上、流量在走廊地板上。

## 第二部分：v3 问题诊断（12 条具体问题）

### 问题 1：金色主题 vs 金融专业感
- **问题描述**：全局色彩体系以金色（oklch gold-300/500）为主色。从 logo、建筑、字体到粒子效果，几乎所有视觉元素都染上了金色。
- **为什么有问题**：金色在金融领域有"土""暴发户""黄金广告"等负面联想。现代金融科技公司的设计语言使用深蓝、钢灰、白色来传达"信任""稳定""精确"。金色破坏了这种专业感，让人联想到澳门赌场或廉价理财产品广告。
- **应该怎样**：主色改为冷色系（深蓝钢或深灰），用一小块品牌色（藏青/电光蓝）做点缀。金色仅仅作为"完成/成功"的状态颜色之一。

### 问题 2：漆黑空洞背景
- **问题描述**：场景背景 `#0e1018` 几乎是纯黑，加上雾效（`FogExp2`），远景完全消失在黑暗中。中央广场空旷，16 棵树分散在圆周上，缺乏空间围合感。
- **为什么有问题**：黑色背景让 3D 场景显得空洞、未完成。用户视角下大部分画面是黑的，只有 5 栋孤立的建筑。这制造了"恐怖游戏"或"太空场景"的感觉，而非金融数字孪生。
- **应该怎样**：室内场景天然不需要"背景"——墙壁和地板就是边界。明亮、围合的空间让用户有"置身其中"而非"漂浮在黑暗里"的感觉。

### 问题 3：建筑外观像玩具积木
- **问题描述**：建筑主体是纯色发光立方体（`BoxGeometry`），每层用水平发光条模拟窗户。屋顶是锥形"王冠"，顶部有旋转金色光环。整体看起来像《我的世界》里的玩具建筑。
- **为什么有问题**：金融行业需要传达"精密""可靠""成熟"，玩具感完全违背这个定位。纯色发光体 + 锥形屋顶 = 魔法城堡，不是金融机构。
- **应该怎样**：放弃建筑外部视角，改为内部视角。办公室场景用真实材质（木纹桌面、玻璃隔断、磨砂金属边框），减少发光，增加环境光照和材质纹理。

### 问题 4：没有角色，场景"无人"
- **问题描述**：v3 场景中没有任何角色。5 栋建筑 + 广场 + 树 + 喷泉 + 数据粒子。场景是"物"的集合，没有"人"的存在。
- **为什么有问题**：NexusAI 的核心价值是"人-任务-数据-流程"的数字孪生。没有角色 = 没有人的映射 = 不完整。Marvis 的小黑马正是通过拟人化角色让"AI 在工作"可视化。v3 缺失了整个"人"的维度。
- **应该怎样**：办公室中必须出现角色——至少是部门 Agent 的化身。金融行业可以使用抽象几何体或精简的职业小人造型，放置于工位/会议室中。

### 问题 5：空间尺度过大，信息密度过低
- **问题描述**：城区地面 80×80 单位，建筑仅 5 栋，分布在 13-16 单位半径的范围。80×80 的地面 95% 是空的。
- **为什么有问题**：空洞 = 用户觉得"没什么可看的"。大空间需要更多细节填充，否则就是浪费屏幕。数字孪生不是模拟城市游戏。
- **应该怎样**：缩小到单个办公室空间（约 20×20 单位），精确到每个工位、每张桌子、每个屏幕。密度带来信息量，信息量带来价值感。

### 问题 6：数据可视化脱离场景
- **问题描述**：KPI 卡片在右侧面板（2D HTML），图表在右侧面板。3D 场景里只有建筑上方漂浮的 Canvas Sprite 标签和粒子的流动线。数据不在场景里。
- **为什么有问题**：用户看 3D 场景需要理解数据，但数据在面板上；看面板又脱离了空间上下文。两者割裂，失去了"数字孪生"的核心意义——空间即数据。
- **应该怎样**：数据嵌入空间。工位桌面上浮 KPI 数值，会议室玻璃墙显示占用/空闲，走廊地板光带颜色表示工作流速度。

### 问题 7：建筑造型缺乏差异化和语义
- **问题描述**：5 栋建筑造型几乎完全相同（BoxGeometry + 锥形屋顶），仅通过颜色区分（金色/橙色/绿色/红色/蓝色）。用户只能靠颜色辨认部门。
- **为什么有问题**：同质化造型让场景缺乏辨识度。建筑外观没有反映部门的"性格"或功能差异。资产管理部和信息技术部的建筑看起来一样，只是换了颜色。
- **应该怎样**：不同部门的办公区域应有不同的空间特征。风控部可以是封闭的"监控室"，投行部可以是开放的"交易大厅"，IT 部可以是密集的"技术工位区"。

### 问题 8：喷泉作为核心装饰元素与金融无关
- **问题描述**：中央广场的核心装饰是一个旋转金色光环喷泉，占据视觉中心位置。
- **为什么有问题**：喷泉 + 金色 + 广场 = 欧洲古典广场设计语言，与金融数字孪生完全没有语义关联。它分散了注意力，制造了"这是什么主题？"的困惑。
- **应该怎样**：室内场景不需要这种装饰。如果有核心视觉锚点，应该是"数据中心核心"或"AI 中枢"——一个与产品功能相关的元素。

### 问题 9：粒子系统过于繁杂
- **问题描述**：有 3 套粒子系统——喷泉粒子（40 个金色粒子）、桥梁数据流粒子（8 对建筑之间的粒子）、环境漂浮粒子（200 个金色粒子）。总计 300+ 个粒子在场景中漂浮。
- **为什么有问题**：过多的粒子让场景看起来"眼花缭乱"，没有信息价值。环境粒子尤其无意义——它们在空气中随机漂浮，不传达任何数据，纯粹是装饰。同时，金色粒子加深了"金色主题"的问题。
- **应该怎样**：大幅精简粒子系统。仅保留有信息价值的数据流——如走廊地板上的光点流动表示工作流进度。每个粒子必须对应真实数据。

### 问题 10：树作为环境元素风格不匹配
- **问题描述**：16 棵低多边形树（褐色圆柱树干 + 绿色球冠）围绕中央广场排列。
- **为什么有问题**：viable 的室外场景元素（树+广场+喷泉）与金融数字孪生的语境不匹配。树的球冠造型进一步加深了"玩具/游戏"感。
- **应该怎样**：室内场景用绿植点缀（参考 Marvis 的办公绿植），但应是现代办公风格——低矮盆栽、细长室内植物，而不是"户外球形树"。

### 问题 11：视角逻辑缺陷
- **问题描述**：默认相机在 26 单位半径处绕中心旋转（auto-orbit），用户可以拖拽旋转和缩放。
- **为什么有问题**：auto-orbit 让场景像个 3D 模型展示器，而非可交互的工作空间。用户始终是"旁观者"而非"参与者"。而且俯视角度让建筑看起来更小、更玩具。
- **应该怎样**：等距/正交视图或第一人称视角，让用户感觉"在办公室里"。相机预设几个关键角度（全局俯视、工位特写、会议室视角），用户切换而非旋转。

### 问题 12：右侧面板与 3D 场景信息重复但无联动
- **问题描述**：右侧面板显示 KPI 卡片和风险列表，左侧面板显示部门树。点击建筑可以飞到该建筑，但点击面板项不会高亮场景中的建筑。
- **为什么有问题**：单向联动（场景→面板）而非双向。用户点击面板中的"投资银行部"时，3D 场景没有任何反应。这削弱了"孪生"的概念——两边应该始终同步。
- **应该怎样**：双向绑定。点击面板项→场景飞到对应区域并高亮。点击场景物体→面板自动展开对应详情。两者同步变化。

## 第三部分：v4 设计方案建议

### 3.1 场景选择

**不做宏大城区，做一个精致的办公室内部。**

| 参数 | 建议 |
|------|------|
| 场景尺寸 | 约 20×15 单位（一个开放办公层 + 1-2 间会议室 + 风控监控区） |
| 视角 | 默认 45° 等距正交视图（类似《模拟人生》视角），干净利落无透视畸变 |
| 空间元素 | 5 个部门工位区（每个 2-4 个工位）+ 1 间玻璃隔断会议室 + 1 个风控监控墙 + 走廊/通道 + 接待/入口区域 |
| 为什么不更大 | 20×15 已经可以放置 20+ 个工位 + 会议室 + 公共区。更大的空间需要更多填充物，但金融数字孪生不需要 100 个工位——需要的是 5 个部门的精华 + 数据密度 |

**具体空间布局建议：**
```
┌──────────────────────────────────────┐
│  [入口/LOGO]                         │
│                                      │
│  [投资银行部]  [资产管理部]          │
│   4个工位       3个工位              │
│                                      │
│  ───── 走廊（数据流光带）─────       │
│                                      │
│  [风险管理部]  [信息技术部]          │
│   风控监控墙    4个工位              │
│                                      │
│  [会议室]       [合规部]             │
│   玻璃隔断      2个工位              │
└──────────────────────────────────────┘
```

### 3.2 色彩体系

**放弃金色，使用现代金融科技配色方案。**

| 角色 | 色值 | 用途 |
|------|------|------|
| **底色（墙/地板）** | `#F1F5F9`（冷白） / `#E2E8F0`（浅灰） | 办公室墙壁、天花板 |
| **表面色（桌面/家具）** | `#FFFFFF` / `#F8FAFC` | 桌面、面板背景 |
| **主色（品牌/交互）** | `#1E3A5F`（深蓝钢） | logo、标题、主要按钮 |
| **辅色（数据/高亮）** | `#3B82F6`（电光蓝）| 图表线条、选中状态、链接 |
| **强调色（成功）** | `#10B981`（翡翠绿）| 正常状态、完成指标 |
| **强调色（警告）** | `#F59E0B`（琥珀色）| 预警状态、中等风险 |
| **强调色（危险）** | `#EF4444`（珊瑚红）| 严重告警、风险指标 |
| **中性色（文字）** | `#1E293B`（深灰）/ `#64748B`（中灰） | 文字层级 |
| **材质色（木纹）** | `#D4A574`（浅橡木） | 桌面、地板 |
| **材质色（玻璃）** | `rgba(255,255,255,0.6)` + `blur` | 玻璃隔断、面板 |

**关键原则**：
- 场景 70% 是冷白/浅灰（墙壁、地板、天花板），让空间明亮通透
- 数据面板用深蓝钢底色 + 白字，与浅色场景形成对比
- 状态颜色严格限于语义用途（绿=正常，琥珀=预警，红=告警）
- 金色仅作为"异常完成"或"里程碑达成"的罕见强调色，占比 < 2%

### 3.3 材质与灯光

**灯光：模拟办公室自然光环境。**

| 灯光 | 配置 | 效果 |
|------|------|------|
| 环境光（Ambient） | `#F8FAFC`, intensity 0.5 | 基础照明，消除全黑阴影 |
| 半球光（Hemisphere） | sky: `#E0F0FF`, ground: `#D0D0D0`, intensity 0.4 | 模拟从窗户进入的天光 |
| 主方向光（Directional） | `#FFF8F0`, intensity 0.6, 从上方 45° 斜射 | 软阴影，模拟天花板灯光 |
| 点光源（Point） | 每个工位上方 1 个，暖白 `#FFF5EB`, intensity 0.3 | 局部照明，增加层次 |

**材质方向：木质 + 磨砂玻璃 + 哑光金属。**

| 物体 | 材质 | 避免 |
|------|------|------|
| 地板 | 浅橡木纹理（`#D4A574` 基色 + subtle noise） | 不要纯色！纹理是消除玩具感的关键 |
| 墙壁 | 哑光白/浅灰，roughness 0.9 | 不要纯白色发光 |
| 桌面 | 中橡木/胡桃木，roughness 0.6 | 不要金属桌面 |
| 玻璃隔断 | 半透明 + 轻微反射，opacity 0.5 | 不要完全透明 |
| 金属边框 | 哑光拉丝铝（#9CA3AF），roughness 0.4, metalness 0.7 | 不要镜面金属 |
| 椅子 | 织物灰（#6B7280），roughness 0.85 | 不要塑料光泽 |

**避免玩具感的核心手段**：
1. **纹理 > 纯色**：地板、桌面必须有纹理贴图（即使低分辨率），纯色 flat shading 是玩具感的元凶
2. **软阴影 > 硬阴影**：使用 PCFSoftShadowMap，柔化边缘
3. **环境光遮蔽**：边角微暗，模拟真实光线的自然衰减
4. **避免发光材质**：不要在场景物体上使用 emissive，数据面板除外
5. **圆角 > 直角**：所有 3D 物体边缘倒角（bevel），直角边缘看起来像初学 3D 作品
6. **比例真实**：桌面厚度 0.05 单位、椅子座高 0.45 单位——比例失真立刻暴露玩具感

### 3.4 角色设计

**金融行业需要介于"纯抽象"和"卡通角色"之间的角色语言。**

| 方案 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| A. 抽象几何体 | 每个部门 Agent 是一个发光的几何体（菱形、八面体等），悬浮在工位上方 | 极简、专业、不冒犯 | 缺乏情感连接 |
| B. 精简职业小人 | 类似 Apple Memoji 的极简风格 —— 圆形头部 + 梯形身体 + 无面部细节 | 有"人"的感觉但不过于写实 | 可能像游戏 |
| C. 数据雕塑 | 角色由数据可视化元素构成 —— 柱体表示负载、光环表示状态 | 前沿、与数据直接关联 | 实现复杂 |
| **D. 推荐方案：方案 A + C 混合** | **每个部门 Agent 是一个几何核心（发光正多面体）+ 外围光环（表示状态）+ 下方数据基础（平台/底座显示 KPI）。不拟人化，但让几何体具有"性格"——通过大小、旋转速度、光环颜色、颤动频率区分不同 Agent。** | 金融级专业感 + 有生命力 + 数据可视化直接附着在角色上 | 需要精细的动画设计 |

**角色状态视觉语言**：

| 状态 | 几何核心表现 | 光环表现 |
|------|------------|---------|
| 正常运转 | 缓慢旋转，发出柔和的电光蓝 | 稳定环形光，低脉冲 |
| 高负载 | 快速旋转，尺寸微增大 | 光环扩展，频率增加 |
| 预警 | 间歇颤动，颜色渐变为琥珀 | 光环断裂，不规则闪烁 |
| 告警 | 明显抖动，颜色变为珊瑚红 | 光环脉冲剧烈，向外扩散波纹 |
| 离线/休眠 | 静止，亮度降低 80% | 光环消失 |

**每个部门对应的几何体**：

| 部门 | 几何体 | 寓意 |
|------|--------|------|
| 投资银行部 | 菱形十二面体 | 多面性、交易结构 |
| 资产管理部 | 截角八面体 | 平衡、组合管理 |
| 风险管理部 | 正二十面体（复杂面>强防护） | 多维度监控 |
| 信息技术部 | 立方体（稳定） | 基础设施 |
| 合规部 | 正四面体（最简稳定结构） | 规则基础 |

### 3.5 数据可视化嵌入

**数据如何"长在"空间里：**

#### A. 工位层级（个人/部门数据）

| 位置 | 数据 | 呈现方式 |
|------|------|---------|
| 桌面屏幕（3D 平面） | 员工姓名、今日任务数、完成率 | 小面板贴附在桌面平面上 |
| 桌面高度 | 工作负载饱和度 | 桌面 Y 轴高度微调（0-0.3 单位浮动），负载越高桌面略微"下沉" |
| 工位上方悬浮粒子 | 当前进行中的工作流 | 2-3 个光点围绕几何 Agent 旋转，每个光点代表一个活跃工作流 |

#### B. 会议室层级

| 位置 | 数据 | 呈现方式 |
|------|------|---------|
| 玻璃隔断颜色 | 占用状态 | 空闲=透明，使用中=电光蓝半透明，超时=琥珀色 |
| 桌面投影 | 当前议题、参会人数、已用时间 | 从上方投影到桌面中央的圆形光斑，内含文字 |

#### C. 走廊/公共区域

| 位置 | 数据 | 呈现方式 |
|------|------|---------|
| 地板光带 | 跨部门工作流速 | 走廊中央一条发光带，粒子沿带流动，流速表示工作流完成速度，颜色（蓝>绿>琥珀>红）表示健康度 |
| 天花板灯带 | 全局系统健康 | 正常=暖白光，局部告警=对应区域灯光变琥珀 |

#### D. 风控监控墙

| 位置 | 数据 | 呈现方式 |
|------|------|---------|
| 大屏幕（3D 平面） | 实时风险指标、告警列表 | 仿真实监控室的屏幕墙，但数据是真实的 2D HTML Canvas 投影到 3D 平面上 |
| 地面红色光圈 | 当前最高告警级别 | 风险等级 1=无光圈，等级 2=琥珀光圈，等级 3=红色脉冲光圈 |

### 3.6 交互设计

**交互模型：探索式浏览 + 精准操作。**

| 交互 | 触发方式 | 行为 |
|------|---------|------|
| 全局浏览 | 鼠标拖拽（左键按住拖动） | 平移场景（pan），非旋转。正交视图下平移更自然 |
| 缩放 | 滚轮 | 连续缩放，范围 0.5× - 3× |
| 聚焦某区域 | 双击工位/会议室/部门 | 相机平滑飞行到该区域，视角变为近距离等距 |
| 返回全局 | 双击空白区域 / Esc 键 | 相机飞回默认全局视角 |
| 悬停信息 | 鼠标悬停 200ms | 浮出 tooltip，显示该物体绑定的关键数据 |
| 选中详情 | 单击物体 | 右侧面板展开该物体的详细数据面板；3D 场景中物体高亮（光晕）|
| 视角预设 | 左上角小按钮组 | [全局] [投行] [资管] [风控] [会议]——一键切换预设相机角度 |
| 时间轴 | 底部时间滑块 | 拖动可查看历史时刻的办公室状态（类似回放） |

**关键交互原则**：
- 所有可交互物体在悬停时有视觉反馈（微发光、微抬起）
- 飞行动画使用 ease-in-out 缓动（建议 cubic-bezier 0.4, 0, 0.2, 1），时长 600-800ms
- 选中物体有明显的视觉标识——不是 v3 的"换个 active 类名"，而是 3D 光晕（glow outline）

沿用 v3 的布局框架（顶部导航 + 左侧列表 + 中间 3D + 右侧面板 + 底部状态栏），但全面刷新视觉：

| 面板 | v3 问题 | v4 改进 |
|------|--------|--------|
| 顶部导航 | 金色 logo | 深蓝钢 logo，极简无渐变 |

## 推荐的最终方案摘要

NexusAI v4 应从一个俯瞰全城的"上帝视角"转变为一个**精致金融办公室内部的等距正交视图**。场景尺寸约 20×15 单位，包含 5 个部门工位区、1 间玻璃会议室和 1 面风控监控墙。色彩体系从金色全面转向冷色调——深蓝钢主色 + 电光蓝辅色 + 冷白底色，状态色严格限于绿/琥珀/红。材质以浅橡木地板、磨砂玻璃隔断、哑光金属边框为主，通过纹理贴图和软阴影消除玩具感。每个部门 Agent 化为悬浮的几何发光体（投资银行=菱形十二面体、风控=二十面体等），通过旋转速度、光环颜色和颤动频率可视化部门状态。KPI 嵌入桌面屏幕和悬浮面板，地板光带表示工作流速度，风控墙模拟真实监控室。交互上，用户平移浏览场景、双击聚焦区域、悬停查看数据，所有操作有 600-800ms 缓动过渡。整个设计的核心精神：**精致**（每个像素都有意图）、**专业**（金融不是游戏）、**明亮**（告别黑暗空洞）、**有生命力**（几何 Agent 的状态律动 + 数据在空间中流动）。
